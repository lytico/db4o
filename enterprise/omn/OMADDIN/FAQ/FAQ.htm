ObjectManager Enterprise FAQ

 

 

General

 

 

What does ObjectManager Enterprise do? Why should I use it?

 

ObjectManager Enterprise is advanced version of the freely available Object Manager. It has features that allow programmers to do all db4o related operations from within Visual Studio .NET IDE. So a programmer can browse a db4o database, query it, access support cases, request for pairing, check latest db4o downloads and do many more things without having to open different applications. This enhances programmer productivity and gives them a nice, new experience of programming with db4o.

 

 

db4o is available in two flavors, what about ObjectManager Enterprise?

 

ObjectManager Enterprise is presently available for .NET programmers as a Visual Studio .NET 2008/2010 plug-in and Eclipse plug-in for Java developers.

 

 

Can I use ObjectManager Enterprise while I debug my application?

 

If your application uses db4o in a client-server mode, then you can use ObjectManager Enterprise (in remote connection mode) while you debug the application. However, if you use db4o in local mode (Db4oFactory.OpenFile()), do not use ObjectManager Enterprise while you debug. This restriction exists because opening two ObjectContainers with Db4oFactory.OpenFile() at the same time is not recommended. This can corrupt the database.

 

Can I start multiple instances of ObjectManager Enterprise on my machine?

 

Yes. It’s possible to start multiple instances of ObjectManager Enterprise on one machine. However, do not connect to the same database in local mode from more one instance.

 

Connecting and Browsing

 

 

How do I connect to a db4o database through ObjectManager Enterprise?

 

ObjectManager Enterprise can connect to db4o in two modes, local and remote. In local mode, browse and select the db4o database file and click connect. In remote mode, enter the host, port, username and password and click connect. In either case, you can even choose from a list of recent connections by clicking it.

 

 

How do I see all classes (along with their attributes) saved in my db4o database?

 

On connecting to a database, a db4o Browser opens and lists all classes saved in the database. User can choose between the Assembly View where all classes are listed in order of Assembly Name or the List View where all classes are listed in a flat view.

 

User can select a class by clicking and expanding the hierarchy by clicking on '+' sign. User can navigate to any attribute at any level in the hierarchy.

 

 

How do I see all objects of a particular class in my database?

 

User can select a class as described above and right click and select Show All Objects from context menu. A new tab displaying all objects of the selected class in a table view opens. To examine, expand and navigate click on an object in the table view. A tab showing details of the selected object in a tree grid view appears below the table.

 

 

Database, Class and Object Properties

 

 

How do I see database size (and other db properties)?

 

A Properties Pane is shown at the bottom of ObjectManager Enterprise window. Click on the Database Properties tab. All database properties such as database size, free space, number of different classes whose objects are saved in the database can be seen here.

 

Note: These properties are not seen for a remote connection.

 

 

How can I find the names of indexed attributes of a class?

 

A Properties Pane is shown at the bottom of ObjectManager Enterprise window. Select a class and click on Class Properties tab in this pane. A table shows all attributes of a class along with access specifier (public, non-public) and index details. This tab also shows the count of objects of the selected class in the database.

 

 

How can I find uuid and version no. of an object in the database?

 

A Properties Pane is shown at the bottom of ObjectManager Enterprise window. Select an object and click on the Object Properties tab in this pane. A table shows uuid and version no. for the selected object.

 

 

 

Ad-hoc Querying

 

 

How to form a query?

 

To build a query, drag and drop attributes of a class from the tree view to the Query Builder pane. Set constraints on attributes by providing a condition and value. Multiple constraints can be joined with logical operators.

 

For complex queries, user can form Parenthesized Query Expressions and join them by logical operators.

 

Click on Run Query to see the results.

 

 

Which elements can participate in a query?

 

Only class attributes of primitive datatype (String, numeric, Boolean, date etc) can participate in a query. These attributes can belong to any level in the reference hierarchy.

 

However, attributes of complex types, collections and primitive elements inside collections cannot participate in the query.

 

 

How to form a nested or parenthesized query?

 

For nested queries, users can form Parenthesized Query Expressions and join them with logical operators. Parenthesized Query Expressions are formed by dragging and dropping attributes from class view to the Query Builder and then associating conditions and providing values.

 

 

 

Updating, Deleting objects

 

 

How to edit and update an object?

 

Editing is possible in both, tree view and table view. Select object to be edited. Edit any attribute value by double clicking on the cell. The new value appears in red and is even reflected in the other view. Click the Save button to update the object in the database.

 

Note: Edit and save one object at a time. Do not edit another object before saving the first.

 

 

How to delete an object?

 

Select the object to be deleted in the table view. Click on Delete button.

 

 

 

Browser functions

 

How can I find the latest db4o versions available for download?

 

Click Tools > ObjectManager Enterprise > db4objects Downloads. This displays a page showing latest available versions of db4o products.

 

 

How can I access professional db4o support?

 

db4o provides professional support to licensed users. Through this, users can raise Support Cases and expect a guaranteed response by email within 24 hours. To access this click Tools > ObjectManager Enterprise > Support Cases or directly click   button on the toolbar.

 

 

What is XTreme Connect? How can I access it?

 

XTreme Connect is the next level of professional support. It allows users to pair up and share his application with a support engineer. The support engineer can look at the user's code and his problem first hand and discuss it over Internet Telephony tools. It’s almost like support through video conferencing.

 

To access XTreme Connect click Tools > ObjectManager Enterprise > XTremeConnect or directly click   button on the toolbar.

 

ShortCut Keys

Details for ShortCut  keys

Press the Alt key on your keyboard. The shourcut keys are visible to user after pressing ALT key. This is a Windows setting.
To avoid pressing ALT key all the time for these shortcut keys, there is a setting to be done by user as follows :
Goto : Control Panel + Display, Appearance tab, Effects, turn off the "Hide the underlined letters" option. after doing this setting, shortcut keys will be visible to the user even without pressing ALT key.

Shortcut keys are available for following features:

After Addin is added to the visual studio. After user has clicked 'connect' button to connect to database following short cut keys can be used:

 

Alt+ O- Opens the database

Alt + C- Cancel opening the database

 

After user gets connected to the database following short cut keys can be used:

 

Alt+ G - Adds a Query Group

Alt + C- Clears the expression group

Alt + R- To run the query formulated in expression group

 

After user runs the query user gets the query result section following shortcut keys can be used:

 

Alt + S- Saves the changes made in results

Alt + D - Deletes the row which has the focus

Alt + Shift + > - helps moving to next page

Alt + Shift + < - helps moving to previous page
